## Problems found (ranked)

1. **Bug — HTML-entity leak in the front-page quick card for Project 04.** `projects.json`'s `shortTitle` for `evidence-review-decision-gate` is `"Debug Memory &amp; Mistake Learning"`. `ProjectCard.tsx` renders `title` through the `<Html>` component (`dangerouslySetInnerHTML`), which correctly decodes `&amp;` → `&` on screen. But `ProofSummary.tsx` renders `shortTitle` as plain JSX text (`<h3>{project.shortTitle ?? project.title}</h3>`, line 14) — React does **not** decode HTML entities in plain text nodes, so the front-page quick card literally shows the six characters `&amp;` instead of `&`. This is real and currently live. Fix: either write `shortTitle` as `"Debug Memory & Mistake Learning"` (no entity — plain text context doesn't need one) or render `shortTitle` through `<Html>` like `title` is. Not fixed yet, pending your approval per "audit first."
2. **Wording gap — Project 02's `shortTitle` drops "Investment."** Full `title` is "Investment Reasoning Learning Database" (correct), but `shortTitle` — used on the front-page quick card — is "Reasoning Learning Database." This matches your complaint exactly; confirmed via grep on the built bundle.
3. **Dead filter machinery still wired into components.** `useFilter.ts`/`FilterBar.tsx` were deleted in v12, but `ProjectCard` and `SupportingSystems` still take an `isMatch` prop and apply `filterable`/`match` classes that can never do anything (nothing sets `body.filtering` anymore — `App.tsx` hardcodes `isMatch={false}` / `noMatch = () => false`). Not visually broken, but it's exactly the "components fighting each other" pattern you're worried about: dead capability left half-wired instead of removed.
4. **~150-180 lines of dead CSS** for the 5 sections deleted in v12 (Operating Model, Fit, Capabilities filter, Lab notebook, Principles) plus the superseded `.core-thinking`/`.hero-thinking` and `.filter-bar` rules. See §8.
5. **No visual "active" state on the front-page quick cards** when their matching project is scrolled to/expanded — only a hover fade. Not a regression, just never built.
6. **Minor copy mismatch**: hero link label is "Visual / architecture portfolio" (lowercase a), not "Visual / Architecture Portfolio ↗" — cosmetic only.

None of these are structural "fighting itself" bugs beyond #1 and #3 — the JSON/component split is clean (§2, §3), the removed-sections cleanup from v12 actually took (§4, §10), and the build is green. The real risk right now is accumulated dead code (§3, §8) plus the two content-accuracy misses (#1, #2), not architectural conflict.

import pageContent from './data/page-content.json'
import projectsData from './data/projects.json'
import type { PageContent, Project } from './data/types'
import { Nav } from './components/Nav'

          ))}

import { useState } from 'react'
import type { Stage } from '../data/types'

export function StageMedia({ stage }: { stage: Stage }) {
  const [errored, setErrored] = useState(false)
  const hasImage = Boolean(stage.image) && !errored
  const src = stage.image ? import.meta.env.BASE_URL + stage.image.replace(/^\//, '') : undefined

  return (
    <div className="stage-media">
      {hasImage ? (
        <img src={src} alt={stage.caption ?? stage.title} onError={() => setErrored(true)} />
      ) : (
        <div className="stage-media-placeholder">Proof capture to be added.</div>
      )}
      {stage.caption && <p className="stage-media-caption mono">{stage.caption}</p>}
    </div>
  )
}
